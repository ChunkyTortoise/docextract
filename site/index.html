DocExtract AI is a document-intelligence stack where a versioned eval corpus, offline replay, and variance-calibrated CI gate are the front door — then two-pass Claude extraction, pgvector search, and an agentic ReAct retrieval loop.

Eval gate, not vibes

Every PR that touches prompts or extraction services runs scripts/eval_offline_replay.py against committed golden responses. The badge on the README is driven by this job — not a one-off notebook score.

  • Deterministic 28-case offline replay on every eval-gated PR
  • Variance-calibrated regression tolerance vs autoresearch/baseline.json
  • Langfuse as the primary observability spine for live traces
  • Ledger-honest metrics in docs/portfolio-metrics.yaml

Then extraction — with agentic RAG

Upload PDFs and images, classify with cost-aware routing, extract structured fields with a two-pass Claude pipeline, embed into pgvector, and query with a ReAct agent that streams reasoning over SSE.

Two-pass extraction

Sonnet draft + verify; Haiku failover behind a circuit breaker. Confidence gating triggers Pass 2 only when needed.

Agentic RAG

Think → Act → Observe loop over hybrid retrieval tools — the primary search story in the live demo and API.

Async + observable

ARQ workers, SSE job progress, Langfuse traces per pipeline run. Prometheus /metrics for ops.

GraphRAG hybrid retrieval is opt-in (GRAPH_RETRIEVAL_ENABLED): regex entity graph, file-backed, not the default path. Semantic cache is implemented but feature-flagged off and not wired into the extraction hot path.
